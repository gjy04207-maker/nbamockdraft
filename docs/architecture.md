# 架构概览

目标：在保证事实准确、规则合规（CBA）、文风贴合的前提下，提供高效写作与可追溯引用的球评生成体验。

## 分层结构
1. 写作工作台（前端）
- 左侧对话：提问、纠错、素材补充
- 右侧预览：生成文章实时更新，支持选中引用并追问
- 引用可视化：段落脚注展示来源（站点/日期/数据表）

2. 编排层（后端 API）
- 统一调度用户意图 -> 数据检索 -> 事实核查 -> 生成
- 明确产出“事实草稿”和“风格化草稿”的链路

3. 数据检索层
- 站点数据：ESPN、nba.com、barttorvik、basketball-excel、basketball-reference、fanspo
- 规则库：CBA PDF 解析后的条款索引
- 个人文风：历史球评索引与摘要

4. 事实与规则校验层
- 事实校验：对引用字段做一致性验证
- 规则校验：涉及薪资、合同、交易条款必须绑定 CBA 依据

5. 生成层（RAG）
- 先检索再生成，强制引用
- 生成内容必须携带可追溯出处

## 关键原则
- 事实优先：事实草稿先于风格化润色
- 透明可追溯：每个关键数据都能点开来源
- 反错机制：当用户描述和数据冲突时提示冲突
