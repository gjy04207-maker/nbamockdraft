# ADR: Mock Draft 数据与交易估值策略

日期：2026-03-18

## 背景
本项目新增模拟选秀功能，需要统一数据来源（Big Board、球队需求、选秀顺序）与交易估值体系，并能在前端稳定呈现。

## 决策
1. **数据抓取与缓存**：采用服务端“定时抓取 + 本地缓存”方案。当前默认回退到内置示例数据；若存在 `data/draft/draft_data.json`，API 优先读取该文件以覆盖默认数据。
2. **数据模型**：统一抽象为 `teams`、`players`、`boards`、`rankings`、`draft_order`、`pick_values` 六类数据；前端只通过 API 获取，不直连第三方网站。
3. **交易估值**：以选秀权价值表为核心，计算送出与接受价值差值，**差值 ≤ 100** 视为可成交，前端据此给出反馈并允许应用交易。

## 影响
- API 增加 `/api/draft/*` 端点作为唯一数据入口。
- 前端模拟选秀、交易系统均依赖该统一数据模型。
- 未来若引入真实数据抓取，仅需在生成 `draft_data.json` 或更新缓存即可。

## 备选方案
- 直接前端抓取第三方页面：因 CORS 与稳定性问题拒绝。
- 完整数据库与爬虫调度：为 v1 过重，暂缓。
