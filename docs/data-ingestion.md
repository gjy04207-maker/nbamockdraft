# 数据接入与更新策略

## 目标
- 小时级新闻与数据更新
- 数据可追溯、可缓存、可回滚
- 遵守站点访问规范与机器人规则

## 数据域
- 比赛与赛程
- 球员基础信息与伤病
- 球队阵容与薪资
- 高级统计（效率、阵地/节奏等）
- 新闻与交易流言

## 站点来源（计划）
- ESPN：新闻、伤病、赛程
- nba.com：官方赛程、球队/球员页面
- barttorvik.com：大学篮球（如需）
- basketball-excel.com：训练与数据教学资料
- basketball-reference.com：历史数据与高级统计
- fanspo.com：薪资与交易机器

## 接入方式（原则）
1. 优先使用官方/公开 API 或可公开访问的数据页
2. 如需爬取：
- 只抓取公开页面
- 严格限速、缓存、增量更新
- 记录抓取时间与版本

## 更新节奏
- 新闻类：每小时抓取一次
- 统计类：每6小时更新一次
- 历史数据：每周增量核对

## 数据存储
- 原始抓取：HTML/JSON 快照
- 结构化表：PostgreSQL
- 向量索引：pgvector

## 合规与风险
- 不绕过登录或付费墙
- 遵守 robots.txt 与站点条款
- 对受限站点预留“接口替代方案”
